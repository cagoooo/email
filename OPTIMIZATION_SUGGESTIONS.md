## ✅ 【階段一】近期已達成里程碑 (Completed)

在最近幾次的迭代中，平台已經完成了許多關鍵的基礎設施與視覺升級：

1. **視覺介面與 RWD 響應式優化**：
   - 首頁 Hero 區塊、跑馬燈與 AI 建議區塊的留白（Padding/Margin）優化，版面更加緊湊洗鍊。
   - Header 導覽列在手機與平板端（窄螢幕）加入帳號長度截斷（`truncate`）機制，解決 RWD 破版問題。
   - 強化 Google OAuth 登入按鈕的互動體驗，新增 Loading 動畫層與狀態文字防卡死機制。
2. **底層架構與效能最佳化 (Performance & Architecture)**：
   - **Bundle Splitting**：在 `vite.config.ts` 中啟用 `manualChunks`，成功分離 `react`, `supabase`, `framer-motion` 與 `ui` 等依賴，有效加快老舊設備的網頁首次載入速度。
   - **Smart/Dumb Pattern**：重構 `TeacherDashboard`, `ParentDashboard`, `Leaderboard` 頁面，將資料獲取 (Data Fetching) 與 UI 元件完美分離，大幅提升未來測試與擴展的便易性。
   - **各種 UI 臭蟲修復**：修復「每日挑戰卡牌」按鈕無法點擊之問題、解決 TypeScript 中的 `Implicit Any` 與 `Babel AST` 類型報錯。
3. **PWA 與快取機制（Service Worker）穩定化**：
   - 修復開發環境下的 SW 攔截問題，確保 Vite HMR（熱更新）不報錯。
   - 確保正式站點發布後，客戶端能順暢取得最新網頁資源。
4. **安全防護與自動化部署 (CI/CD)**：
   - 完善 GitHub Actions 工作流 `deploy.yml`，結合 `__PLACEHOLDER__` 機制注入環境變數並執行 API Key 安全掃描，落實儲存庫零洩漏。

## 🏃 【階段二】下一波體驗升級：深挖「互動感與持續學習」 (1-2 週內)

這部分主要針對使用者介面與既有功能的深度拋光。

### 1. 🌓 暗色模式 (Dark Mode) 全域切換
*   **現狀**：系統已安裝 `next-themes` 且底層元件支援深色模式，但缺乏切換入口。
*   **具體作法**：在頂部導覽列 (Header) 加入一個帶有動態轉換效果的「日月切換按鈕」，讓學生在家或夜間使用時更保護眼睛，並儲存偏好至 `localStorage`。

### 2. 🛡️ 實作防護機制 (Anti-Cheat Server-side Validation)
*   **現狀**：目前送出分數、遊戲結束或解鎖成就的規則較依賴前端運算，有被同學修改本地參數的潛在風險。
*   **具體作法**：利用 Supabase Edge Functions 實作「驗證 API」，前端僅上交「答對題數、挑戰耗時」；由後端決定是否發放點數與徽章，徹底防護點數作弊問題。

### 3. 🧩 進一步的 Route 路由延遲載入 (Lazy Loading)
*   **現狀**：雖然已將函式庫 (Vendor) 打包分離，但所有頁面仍在首次一併被載入。
*   **具體作法**：透過 React `lazy` 與 `Suspense` 自動對 `/admin`, `/teacher`, `/parent` 等後台管理路由進行切片，確保一般學生載入網頁時「不需要下載老師用的程式碼」。

## 🚀 【階段三】核心擴充：強化社交與個人化學習 (未來 1 個月內)

這是能大幅提昇學生「學習動機與黏著度」的關鍵功能藍圖。

### 4. 🤖 升級 AI 學習助理 (串接真實 Gemini)
*   **概念**：目前的 AI 建議是前端預設靜態字串。可升級為呼叫真正的 Google Gemini AI 生成個人化鼓勵對話。
*   **具體作法**：
    *   在 Firebase/Supabase Functions 建立一支安全 API（避免前端暴露 API Key）。
    *   將學生當前的學習弱點（例如：密碼強度測驗錯兩次）送給 `gemini-2.5-flash-lite` 模型處理。
    *   回傳具有童趣、專屬該學生的生動回饋語句（如：「小明！你的密碼越來越難被破解了，繼續保持！」）。

### 5. 🏫 班級對抗與即時排行榜 (Supabase Realtime)
*   **概念**：遊戲目前的競爭機制是靜態的。加上「全服即時廣播」會大幅提升刺激感。
*   **具體作法**：
    *   啟用 Supabase Realtime 訂閱。
    *   當任何同學解鎖困難成就或是累積獲得大量積分時，全網畫面的右下角會升起即時跑馬燈（「🔥 恭喜 三年甲班 王小明 剛獲得 1000 分！」）。
    *   大螢幕長條圖會隨著答題動態攀升，創造運動會般的極致熱鬧感。

### 6. ⚔️ 多人連線 PK 模式 (1 對 1 知識問答)
*   **概念**：讓學生之間可以發起對戰挑戰。
*   **具體作法**：
    *   設計類似「Kahoot!」的小型對戰大廳。
    *   兩人連線後，同屏回答 5 題 Email 或資安題，比拼速度與正確率。
    *   贏家可贏得少量的對手點數，甚至有專屬的「PK 勝利星星」成就徽章。

### 7. 🎁 點數商城 (Shop) 獎勵深度化
*   **概念**：有了學習點數就需要有耗材，增強蒐集誘因。
*   **具體作法**：
    *   除現有的「頭像框」、「名字顏色」外，擴展販售「全站背景主題」（如：賽博龐克風、星空宇宙風），讓網站外觀高度個人化。
    *   甚至可以販售「特殊滑鼠游標特效」（如：拖曳時會有魔法軌跡）。

## 📈 【階段四】永續發展：教育現場實踐 (未來 1-2 個月外)

### 8. 🧾 學習成效自動匯出 (PDF/Excel 報表)
*   **挑戰**：減輕老師負擔，讓數據可視化成為實體報告。
*   **解決方案**：
    *   在 `TeacherDashboard` 加入報表下載功能。
    *   使用 `@react-pdf/renderer` 或前端直接轉 Excel，老師一鍵就能將「全班 Email 學習達成率」、「資安錯誤分析」匯出列印，便於學校進行資訊教育成果驗收。

### 9. 📝 動態教師題庫系統
*   **挑戰**：目前的題目與關卡可能綁定於程式碼之中，不夠靈活。
*   **解決方案**：
    *   提供教師一個專屬的題庫編輯後台介面。
    *   老師可以依據近期校內活動（如：網安宣導週）靈活新增「反詐騙知識」、「網路霸凌防範」等時事題目。
    *   題目直接存入資料庫，實現平台的無痛長遠維護。
